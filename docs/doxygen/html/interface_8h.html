  • ask for an action
  • ask for machine information for creation (see Machine)
  • ask for machine information for update/delete
  • ask for Staff information

void interface_showMap(const Map* map)

Show the map.

Parameters
map in a map

We are waiting for a grid, like

     0     1
  +----++----+
0 | M  |     |
  +----++----+

Where 0,1 are the column indexes and 0 is the row index. M is the first letter for machine, but well no need to write a lot, just enough so that the player can guess whether he would want to check this case or not.

At the end, we want to find basic information such as

  • E value
  • DD value
  • which turn is it
  • number of FISE
  • number of FISA
  • the production mode
  • the number of hired staffs (or their names followed by the count if different of one)

void interface_showStaffList(const Map* map)

Parameters
map used to fetch staff bought

Should take a map and print the list of the staff. Please do take note that this method use a global variable containing all data about the staff.

This method should be a catalog of available staffs. You must add right after a staff name the count of this staff that the user have already bought.

--> ({n°}) Name E:{cost} DD:{cost} Desc:"perks"
...

Precondition

staff list must be set

map must contains the list of staff that the player bought

Action interface_chooseAction()

Ask for the user action.

Returns a valid action chosen by the user.

Ask for the user action Should be a valid action, if not ask again.

Actions returned should be implemented by

  • ACTION_SHOW_MAP: interface_showMap
  • ACTION_CANCEL_ACTION: no function, can't be used here ! (only when an action got chosen)
  • ACTION_EXIT : caller should ends the program
  • ACTION_LIST_ACTIONS : call interface_list_actions

  • ACTION_END_TURN : next turn, call of map_endTurn

  • ACTION_HIRE_FISE : hire fise, call of map_hireFISE
  • ACTION_HIRE_FISA : hire fisa, call of map_hireFISA

  • ACTION_CHANGE_FISA_MODE : change fisa production mode, call of map_changeProductionFISA

  • ACTION_LIST_MACHINES : should call interface_showMachinesList
  • ACTION_BUY_MACHINE : should ask interface_askAddMachine and interface_askMachineLocation then call map_addMachine
  • ACTION_BUY_STAFF : should ask interface_askBuyStaff then map_buyStaff

  • ACTION_ASK_STAFF_LIST : should call interface_showStaffList

  • ACTION_UPGRADE_MACHINE : interface_askMachineLocation then map_upgradeMachine
  • ACTION_DESTROY_MACHINE : interface_askMachineLocation then map_destroyMachine

Machine* interface_askAddMachine()

ask user machine specific details

Returns the machine that the user wanted to add.

Precondition

this method should be called after (or before) asking the user where the machine should be put

If the user wants to go back, then he can using ACTION_CANCEL_ACTION action.

We must fill each value inside our machine, for that check Machine type.

Vector2D interface_askMachineLocation()

Ask for machine location.

Returns a two dimensions vector that should contains a positive x and a positive y.

Ask for machine location, this function should be used when the user triggered upgrade/delete machine action

The programmer won't have to check whether this location is a valid machine location, that would have to be done by the one calling it.

If the user wants to go back, then he can using ACTION_CANCEL_ACTION action.

Staff interface_askBuyStaff()

Ask which Staff the user wants to buy.

Returns a valid Staff that the user chosen. Or null if he asked to cancel.

The user input an id (that he could find using list staff action) and we returns the staff in the global staff array that match the given id.

If the user wants to go back, then he can using ACTION_CANCEL_ACTION action.
